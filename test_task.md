# Тестовый план по написанию валидатора номера телефона

## Цель
Разработать валидатор номера телефона, который проверяет корректность формата телефонного номера и соответствие его стандартам.

## Основные требования к валидатору
- Валидатор должен принимать строку в качестве входных данных
- Поддержка международного формата (+7XXXXXXXXXX, +1XXXXXXXXXX и т.д.)
- Проверка на допустимую длину номера (обычно 10-11 цифр без кода страны)
- Проверка форматов: с разделителями (-, ), пробелами и без них
- Форматы, которые должны проходить валидацию: 
  - +7(926)123-45-67
  - +7 926 123-45-67
  - +7-926-123-45-67
  - +79261234567
  - 8(926)123-45-67
  - 8-926-123-45-67
  - 8 926 123 45 67
  - (926)123-45-67
  - 926-123-45-67
  - 926.123.45.67
  - 9261234567

## Тестовые сценарии

### Позитивные тесты
1. Проверить валидацию номера в формате +7(926)123-45-67
2. Проверить валидацию номера в формате +7 926 123-45-67
3. Проверить валидацию номера в формате +7-926-123-45-67
4. Проверить валидацию номера в формате +79261234567
5. Проверить валидацию номера в формате 8(926)123-45-67
6. Проверить валидацию номера в формате 8-926-123-45-67
7. Проверить валидацию номера в формате 8 926 123 45 67
8. Проверить валидацию номера в формате (926)123-45-67
9. Проверить валидацию номера в формате 926-123-45-67
10. Проверить валидацию номера в формате 926.123.45.67
11. Проверить валидацию номера в формате 9261234567

## Технические требования
- Валидатор должен быть реализован в виде функции validate_phone_number(phone_string)
- Функция должна возвращать True при корректном номере, False при некорректном
- Использовать регулярные выражения для проверки формата
- Обработка исключений для некорректных типов данных
- Документирование кода

## Критерии успеха
- Все позитивные тесты возвращают True
- Все негативные тесты возвращают False
- Валидатор обрабатывает все указанные форматы
- Отсутствие ошибок выполнения при граничных условиях
- Покрытие кода тестами не менее 80%
