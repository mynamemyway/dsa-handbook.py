## Сортировки

Процесс упорядочивания элементов по определённому критерию.

### Встроенная в Python `.sort()` (Timsort)

**Гибридная природа:** Она использует Merge Sort для больших кусков, но переключается на Insertion Sort, когда куски становятся маленькими (около 32-64 элементов).

**Почему?** Потому что Insertion Sort побеждает сложные алгоритмы на малых данных.

### Хак 2: Частично отсортированные данные

Если в задаче сказано: *«У вас есть отсортированный массив, но 2 элемента перепутаны»* или *«Элементы находятся не дальше k позиций от их цели»*, используйте Insertion Sort.

Он будет работать за O(n) или O(nk), что очень быстро.

### Хак 3: Стабильность (Stability)

Стабильность означает, что если у вас есть два объекта с одинаковым значением (например, два человека с именем «Боб»), их исходный порядок сохраняется.

## Тема 15 - Основы сортировки (медленные алго)

### Selection Sort (Сортировка выбором) 🔍

**Концепция:** Ищите во всем неотсортированном списке самое маленькое число (минимум) и меняйте его местами с началом.

**Как это работает:**

1. Начните с индекса 0. Сканируйте от 0 до end. Найдите минимум.
2. Поменяйте этот минимум с индексом 0.
3. Начните с индекса 1. Сканируйте от 1 до end. Найдите минимум.
4. Поменяйте этот минимум с индексом 1.

**Зачем использовать?** Это минимизирует количество Обменов (Swaps). Bubble sort меняет элементы постоянно; Selection sort меняет только один раз за проход. Это полезно, если «запись в память» стоит дорого (например, флэш-память).

**Пример реализации:**

```python
def selection_sort(arr):
    n = len(arr)

    # Проходим по списку
    for i in range(n - 1):
        min_index = i

        # Ищем индекс самого маленького элемента в оставшейся части
        for j in range(i + 1, n):
            if arr[j] < arr[min_index]:
                min_index = j

        # Если нашли элемент меньше текущего, меняем их местами
        if min_index != i:
            # Меняем значения местами
            arr[i], arr[min_index] = arr[min_index], arr[i]

    return arr

# Пример использования:
my_list = [64, 25, 12, 22, 11]
sorted_list = selection_sort(my_list)
print(sorted_list)
# Вывод: [11, 12, 22, 25, 64]
```

### Bubble sort (Сортировка пузырьком)

Работает по принципу «всплытия» самого большого элемента в конец списка на каждой итерации.

**Алгоритм:**

1. Сравните индекс 0 и 1. Если Left > Right, обменяйте.
2. Перейдите к 1 и 2. Повторите.
3. После одного полного прохода самое большое число гарантированно окажется в самом конце.
4. Игнорируйте последний элемент (он отсортирован) и повторите для остальных.

**Пример трассировки:** `[5, 3, 8, 4, 2]`

* Проход 1:
  * `[5, 3]` → Swap → `[3, 5, 8, 4, 2]`
  * `[5, 8]` → OK
  * `[8, 4]` → Swap → `[3, 5, 4, 8, 2]`
  * `[8, 2]` → Swap → `[3, 5, 4, 2, 8]` *(8 теперь зафиксировано в конце)*

```python
def bubble_sort(arr):
    n = len(arr)  # Узнаем длину списка

    # Внешний цикл: сколько раз нужно пройти по списку
    for i in range(n - 1):
        # Внутренний цикл: сравниваем соседние пары
        # С каждым шагом i «хвост» списка уже отсортирован
        for j in range(n - 1 - i):
            # Сравниваем соседние элементы
            if arr[j] > arr[j + 1]:
                # Меняем их местами, если левый больше правого
                arr[j], arr[j + 1] = arr[j + 1], arr[j]

    return arr
```

**Главные фишки:**

1. `n - 1 - i`: Это важная деталь. Мы вычитаем `i`, потому что после каждого прохода самый большой элемент в оставшейся части гарантированно оказывается в конце, и его больше не нужно проверять.
2. **Сравнение соседей:** В отличие от сортировки выбором (где мы искали минимум), здесь мы просто «двигаем» большие числа вправо через обмены с соседом.

**Пример с оптимизацией:**

```python
def bubble_sort_optimized(arr):
    n = len(arr)

    for i in range(n - 1):
        # Флаг: были ли перестановки в этом проходе?
        swapped = False

        for j in range(n - 1 - i):
            if arr[j] > arr[j + 1]:
                # Меняем местами, если порядок неверный
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True  # Фиксируем, что перестановка была

        # Если за весь проход ни разу не сработал swap — список готов
        if not swapped:
            break

    return arr
```

### Insertion Sort (Сортировка вставками) 🃏

**Концепция:** Сортируйте список так же, как вы сортируете игральные карты в руке. Вы берете новую карту и сдвигаете её назад в правильное место среди карт, которые вы уже держите.

**Как это работает:**

1. Предположим, элемент 0 отсортирован.
2. Возьмите элемент 1. Сравните с 0. Если меньше, поменяйте.
3. Возьмите элемент 2. Сдвигайте его влево, пока он не встанет на место.

**Суперсила:** Он **Адаптивный**. Если список почти отсортирован, Insertion Sort работает почти за время O(n). Это невероятно для «онлайн-алгоритмов», где данные поступают потоком.

---

**Реализация:**

```python
def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1

        # Сдвигаем элементы arr[0..i-1], которые больше key,
        # на одну позицию вперед
        while j >= 0 and key < arr[j]:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
    return arr
```

## Divide and Conquer (Разделяй и властвуй)

Это принцип (парадигма) построения алгоритмов.

1. Разбиваем одну большую и сложную задачу на несколько маленьких.
2. Решаем их по отдельности.
3. Склеиваем результаты.

**Сортировки, работающие по этому принципу:**

* **Merge Sort** (Сортировка слиянием) — самая наглядная реализация этого принципа.
* **Quick Sort** (Быстрая сортировка / Сортировка Хоара).

---

## Быстрая сортировка

1. Выбираем рандомный pivot.
2. Делим массив на 2.
3. В левый элементы < pivot, в правый > pivot.
4. Делим подмассивы и повтор рекурсивно.
5. Склеиваем все подмассивы.

## Тема 16: Быстрые сортировки

### Quick Sort (Быстрая Сортировка) ⚡️

Quick Sort тоже делит список, но делает основную работу до разделения, используя **Pivot** (Опорный элемент).

**Алгоритм:**

1. **Выбор Pivot:** Выберите одно число из массива (например, последнее).
2. **Partition (Разбиение):** Это магический шаг. Переставьте массив так, чтобы:
   * Все числа меньше Pivot ушли Влево.
   * Все числа больше Pivot ушли Вправо.
3. **Рекурсия:** Теперь Pivot находится на своем идеальном, финальном месте. Повторите процесс для левой и правой части.

**Проблема Pivot:** ⚠️

Если вы каждый раз выбираете самое маленькое или самое большое число в качестве pivot (например, сортируя уже отсортированный список), Quick Sort деградирует до O(n²).

* **Решение:** случайный pivot или медиану из 3 чисел.

**LeetCode:**

```python
def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    else:
        pivot = arr.pop()
        greater = []
        lower = []

        for item in arr:
            if item > pivot:
                greater.append(item)
            else:
                lower.append(item)

        return quick_sort(lower) + [pivot] + quick_sort(greater)
```

### LeetCode Хак: Quick Select 💡

Это скрытая жемчужина.

**Задача:** «Найти K-й самый большой элемент в массиве.»

* **Наивно:** Отсортировать массив O(n log n) и взять индекс k.
* **Pro (Quick Select):** Используйте логику Partition из Quick Sort.

**Логика:**

Если вы разбили (Partition) массив, и Pivot приземлился на индекс k, поздравляем! Вы нашли K-й элемент, не сортируя остальную часть массива.

* **Сложность:** O(n) среднем.

---

**Дополнительно:**

```python
def quick_sort(arr):
    # Базовый случай: если в списке 0 или 1 элемент, он уже отсортирован
    if len(arr) <= 1:
        return arr

    # Находим индекс опорного элемента (pivot) посередине
    pivot_index = len(arr) // 2
    pivot = arr[pivot_index]

    left = []
    right = []

    # Распределяем элементы по двум спискам
    for i in range(len(arr)):
        # Пропускаем сам опорный элемент по его индексу
        if i == pivot_index:
            continue

        if arr[i] < pivot:
            left.append(arr[i])  # Все, что меньше опорного — влево
        else:
            right.append(arr[i])  # Все, что больше или равно — вправо

    # Рекурсивно сортируем части и склеиваем: лево + опорный + право
    return quick_sort(left) + [pivot] + quick_sort(right)
```

**Через генераторы:**

```python
def quick_sort_short(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[len(arr) // 2]

    left = [x for i, x in enumerate(arr) if x < pivot and i != len(arr)//2]
    right = [x for i, x in enumerate(arr) if x >= pivot and i != len(arr)//2]

    return quick_sort_short(left) + [pivot] + quick_sort_short(right)
```

### Merge Sort (Сортировка Слиянием) 🧩

**Стратегия:** Разделяй и Властвуй (Divide and Conquer).

Она полагается на простой факт: очень легко слить два списка, которые уже отсортированы.

**Алгоритм:**

1. **Divide (Разделение):** Разрежьте список пополам. Затем разрежьте эти половины пополам. Продолжайте резать, пока в каждом списке не останется только 1 элемент. *(Список из 1 элемента технически «отсортирован»)*.
2. **Conquer (Слияние):** Возьмите два маленьких отсортированных списка и соедините («сшейте») их вместе в один большой отсортированный список.
3. **Repeat (Повторение):** Продолжайте сливать вверх, пока не получите один большой отсортированный список.

**Зачем использовать?** Она **Стабильна (Stable)** 🔒

Если вы сортируете список товаров по Цене, а затем по Рейтингу, Merge Sort сохранит порядок Цен для товаров с одинаковым Рейтингом. Именно поэтому `.sort()` в Python использует вариацию Merge Sort (называемую **Timsort**).

```python
def merge_sort(arr):
    if len(arr) > 1:
        mid = len(arr) // 2
        left_half = arr[:mid]
        right_half = arr[mid:]

        # Рекурсия: Сортируем половины
        merge_sort(left_half)
        merge_sort(right_half)

        # Шаг слияния: Собираем обратно
        i = j = k = 0
        while i < len(left_half) and j < len(right_half):
            if left_half[i] < right_half[j]:
                arr[k] = left_half[i]
                i += 1
            else:
                arr[k] = right_half[j]
                j += 1
            k += 1

        # Добавляем остатки...
        while i < len(left_half):
            arr[k] = left_half[i]
            i += 1
            k += 1

        while j < len(right_half):
            arr[k] = right_half[j]
            j += 1
            k += 1

    return arr
```

### Сравнение: Merge Sort vs Quick Sort

| Характеристика       | Merge Sort                          | Quick Sort                              |
|----------------------|-------------------------------------|-----------------------------------------|
| **Стратегия**        | Сначала раздели, потом слей         | Сначала отсортируй (Partition), потом раздели |
| **Сложность (Время)**| O(n log n) (Всегда)                 | O(n log n) (Среднее), O(n²) (Худшее)   |
| **Сложность (Память)**| O(n) (Нужна доп. память)            | O(log n) (Мало памяти)                  |
| **Где используется** | Python, Java (Объекты)              | C++, Java (Примитивы)                   |

## Связный список (односвязный)

Для добавления или удаления нужно только менять ссылки.

* **Singly Linked List** — односвязный список (каждый узел ссылается только на следующий).
* **Doubly Linked List** — двусвязный список (у каждого узла есть ссылка и на следующий, и на предыдущий).
* **Circular Linked List** — кольцевой.

### Сложность связных списков

Связные списки хорошо подходят для задач, когда мы часто удаляем/добавляем элементы в начало/конец и не так часто в середину.

**На питоне:**

```python
# Класс для создания отдельного элемента (узла) списка
class Node:
    def __init__(self, value):
        self.value = value  # Само данное (число, строка и т.д.)
        self.next = None    # Ссылка на следующий узел
        self.prev = None    # Ссылка на предыдущий узел (делает список двусвязным)

# Класс для управления всем списком
class DoublyLinkedList:
    def __init__(self):
        # В начале список пуст, поэтому головы и хвоста нет
        self.head = None
        self.tail = None

    # Метод добавления элемента в конец (хвост) списка
    def append(self, value):
        # 1. Создаем новый объект узла из переданного значения
        new_node = Node(value)

        # 2. Проверяем, пуст ли список (нет ли в нем "головы")
        if not self.head:
            # Если пустой, то новый узел становится одновременно и головой, и хвостом
            self.head = new_node
            self.tail = new_node
        else:
            # 3. Если в списке уже есть элементы:
            #    У текущего хвоста ставим ссылку "вперед" на наш новый узел
            self.tail.next = new_node

            # У нового узла ставим ссылку "назад" на текущий хвост
            new_node.prev = self.tail

            # 4. Переносим метку "хвост" на наш новый узел (теперь он последний)
            self.tail = new_node

# Пример работы:
list_example = DoublyLinkedList()
list_example.append("Первый")
list_example.append("Второй")

print(f"Первый элемент: {list_example.head.value}")
print(f"Второй элемент: {list_example.head.next.value}")
```

### База «процедурного» подхода к спискам, где мы работаем напрямую с узлами.

#### 1. Обход списка (Печать)

«Поставить палец на Голову». Мы используем цикл `while`, потому что не знаем заранее, сколько в списке узлов.

```python
def print_list(head):
    curr = head  # Ставим "указатель" на начало

    while curr is not None:  # Пока не упремся в пустоту (конец списка)
        print(curr.val)      # Печатаем данные текущего узла
        curr = curr.next     # ПЕРЕХОД: перемещаем указатель на следующий адрес в Куче

    print("Конец списка!")
```

#### 2. Вставка в начало (Правильный порядок)

Критически важна последовательность. Если сначала изменить `head`, мы потеряем адрес всей остальной цепочки.

```python
def insert_at_front(head, new_val):
    new_node = ListNode(new_val)  # 1. Создали новый объект

    # 2. Сначала привязываем НОВЫЙ узел к СТАРОМУ началу.
    #    Теперь новый узел "держит" всю остальную цепь.
    new_node.next = head

    # 3. И только теперь объявляем новый узел официальным началом (head).
    head = new_node
    return head
```

#### 3. Разбор "Плохого кода" (Почему это ловушка)

Если порядок нарушен:

```python
new_node = ListNode(99)
head = new_node        # ОШИБКА: Мы забыли, где был старый список.
new_node.next = head   # ОШИБКА: Узел зациклился на себе.
```

**Результат:** Все старые данные (1 → 2 → 3) остались в Куче, но ссылок на них больше нет. Сборщик мусора Python их удалит. Ты «отрезал» голову от туловища.

#### 4. Секретное оружие: Dummy Node (Фиктивный узел)

Это концепция, которой часто не хватает для уверенности. Зачем нам узел со значением -1, которое «не важно»?

**Проблема:** Когда ты удаляешь или вставляешь узел, тебе всегда нужно знать «предыдущий» (`prev`). Но у самого первого узла (головы) нет предыдущего. Это заставляет писать кучу `if head is None`.

**Решение:** Мы создаем «подставного» шпиона перед настоящим списком.

```python
# Dummy Node — это предохранитель
dummy = ListNode(-1)   # Создаем пустой узел-заглушку
dummy.next = head      # Привязываем его ПЕРЕД настоящим началом

# ТЕПЕРЬ:
# У самого первого реального узла (head) ТОЖЕ есть предыдущий — это dummy.
# Нам больше не нужны проверки "а не пуст ли список?",
# потому что dummy всегда существует.

# В конце работы мы просто возвращаем:
return dummy.next  # Это и есть наше реальное начало списка
```

**Итог по твоим примерам:**

1. **Индексов нет.** Мы работаем только с объектами и ссылками `.next`.
2. **Безопасность:** Сначала связывай новый узел с хвостом, а потом перенаправляй голову.
3. **Dummy Node:** Используй его, чтобы код стал чище и не раздувался от проверок `if`.

> Dummy Node никогда не несет полезных данных, его роль — просто "подержать дверь", чтобы алгоритм не сломался на пустом списке.

## Паттерн указателей

### Быстрый и Медленный Указатели (Черепаха и Заяц) 🐢🐇

Это паттерн, используемый в 30% задач на связные списки. У вас есть два указателя, движущихся с разной скоростью.

* **Медленный указатель (slow):** Движется на 1 шаг за раз (`slow = slow.next`).
* **Быстрый указатель (fast):** Движется на 2 шага за раз (`fast = fast.next.next`).

**Сценарии использования:**

1. **Поиск Середины:** Когда `fast` (Заяц) достигает финиша, `slow` (Черепаха) будет ровно на полпути.
2. **Обнаружение Цикла:** Если список зацикливается на себе, быстрый бегун в конечном итоге обгонит медленного бегуна на круг и догонит его сзади. Если они когда-нибудь встретятся, значит, есть цикл.

### Стратегия LeetCode: "LRU Cache" 🧠

Самая известная задача на собеседованиях с использованием Двусвязных Списков — это **LRU Cache** (Least Recently Used).

**Проблема:** Разработайте структуру данных, которая хранит пары Ключ-Значение. У нее есть максимальный размер. Если она переполняется, удалите элемент, который использовался давнее всего.

**Решение:**

Нам нужен быстрый поиск (O(1)) И быстрое переупорядочивание (O(1)).

* **Hash Map:** Хранит Key → Node. (Для мгновенного доступа).
* **Doubly Linked List:** Хранит порядок. Head = Старый, Tail = Новый.

**Алгоритм:**

1. **Get(Key):** Найти узел в Карте. Отсоединить его от списка. Переместить его в Конец (Front/Tail).
2. **Put(Key, Val):**
   * Если ключ существует: Обновить Val, переместить в Конец.
   * Если ключ новый: Создать узел, добавить в Конец.
   * Если Полон: Удалить узел в Голове (Самый старый), удалить из Карты.

## Что такое Стек?

Стек — это линейная структура данных, которая следует принципу **LIFO: Last-In, First-Out** (Последним пришел — первым ушел).

Это означает, что элемент, который попадает в стек последним, покидает его самым первым. Это полная противоположность Очереди (Queue).

### Аналогия: Стопка Тарелок 🍽️

Представьте пружинный диспенсер для тарелок в столовой.

1. **Push (Толкнуть):** Вы кладете новую чистую тарелку Сверху.
2. **Pop (Вытолкнуть):** Когда клиенту нужна тарелка, он берет её Сверху.
3. **Ограничение:** Вы не можете взять нижнюю тарелку. Если попытаетесь, вся стопка упадет. Вы должны сначала убрать верхние.

---

### Почему они такие быстрые?

Потому что вы никогда не сдвигаете элементы (как при вставке в Массив). Вы просто кладете элемент сверху или снимаете его.

---

### Как использовать Стек в Python 🐍

Вам не нужно импортировать специальную библиотеку. Встроенный `list` в Python — это и есть замаскированный Стек!

**Полный код примера со скрина:**

```python
stack = []

# 1. PUSH (Добавить наверх)
# В Python "конец" списка — это "верхушка" стека.
stack.append(10)
stack.append(20)
stack.append(30)

# Стек выглядит так: [10, 20, 30] (30 наверху)

# 2. PEEK (Посмотреть наверх)
# Мы обращаемся к последнему элементу используя индекс -1
print(stack[-1])  # Вывод: 30

# 3. POP (Удалить сверху)
top_item = stack.pop()
# Возвращает 30. Стек теперь [10, 20]

# 4. Проверка на пустоту
if not stack:
    print("Стек пуст!")
```

### Когда использовать Стек?

Если вы слышите слова **"Реверс"**, **"Вложенный"**, **"Отмена (Undo)"** или **"Самый недавний"**, ваш мозг должен кричать **СТЕК**.

#### 1. Кнопка "Отмена" (Ctrl+Z) 🔙

Каждый раз, когда вы печатаете слово в редакторе, это состояние Push-ится в стек.

* Состояние 1: "Привет"
* Состояние 2: "Привет Мир"
* Пользователь жмет Ctrl+Z
* Действие: Pop Состояния 2.
* Текущее состояние: "Привет".

#### 2. История Браузера 🌍

* Вы заходите в Google → Push "Google".
* Вы заходите в YouTube → Push "YouTube".
* Вы жмете "Назад" → Pop "YouTube" (Возврат в Google).

#### 3. Стек Вызовов Функций (Recursion) 📞

Именно так компьютер выполняет код. Когда Функция А вызывает Функцию Б, компьютер "ставит на паузу" А и кладет её в стек. Он выполняет Б. Когда Б завершается, он достает (**pop**) А обратно, чтобы продолжить работу.

> **Заметка:** Именно поэтому бесконечная рекурсия вызывает ошибку "Stack Overflow" (Переполнение стека)!

### Паттерн LeetCode: «Monotonic Stack» (Монотонный Стек) 📉

Это продвинутый паттерн, используемый в задачах уровня "Medium" и "Hard" (например, *Daily Temperatures*).

**Концепция:**

Вы поддерживаете стек отсортированным (по возрастанию или убыванию), удаляя элементы, которые нарушают порядок. Это позволяет мгновенно находить нужные элементы (например, следующий больший или меньший).

**Пример: Найти следующий больший элемент**

Вход: `[2, 1, 5]`

1. Стэк: `[2]`
2. Следующий 1. 1 > 2? Нет. Push 1. Стэк: `[2, 1]`.
3. Следующий 5. 5 > 1? Да! Pop 1 (5 — следующий больший для 1).
4. 5 > 2? Да! Pop 2 (5 — следующий больший для 2).
5. Push 5.

---

### 🧠 Как это связано с твоим гайдом?

* Этот паттерн (**Monotonic Stack**) — классическая схема для поиска границ (Next Greater Element).
* Вместо того чтобы делать двойной цикл (O(N²)), ты используешь стек один проход (O(N)), поддерживая его "монотонным".
* Это прямо отвечает на вопрос из гайда: «Алгоритмы поиска: как быстро найти нужное».

## Что такое Очередь?

Очередь (Queue) — это линейная структура данных, которая следует принципу **FIFO: First-In, First-Out** (Первым пришел — первым ушел).

Это полная противоположность Стеку. В Стеке (**LIFO**) последним обслуживается тот, кто пришел последним. В Очереди первым обслуживается тот, кто пришел первым. Это делает Очереди «справедливыми».

### Аналогия: Очередь в Кофейне ☕

Представьте, что вы в Starbucks.

1. **Enqueue (Встать в очередь):** Вы приходите и встаете в Конец (Back) очереди.
2. **Wait (Ожидание):** Вы ждете, пока обслужат всех перед вами.
3. **Dequeue (Обслуживание):** Бариста вызывает человека из Начала (Front).
4. **Справедливость:** Если вы попытаетесь влезть в середину, люди разозлятся. Если бариста проигнорирует первого человека и обслужит последнего (поведение Стека), начнется бунт.

---

### "Ловушка Python" ⚠️🚨

Это ошибка №1, которую совершают новички на собеседованиях по Python. У вас может возникнуть соблазн использовать простой `list` (список) в качестве очереди.

**НЕ ДЕЛАЙТЕ ЭТОГО.**

```python
# ❌ ПЛОХОЙ КОД (Не используйте на интервью)

queue = []

queue.append(10)   # Enqueue: O(1) - Быстро
queue.append(20)

# Проблема: Удаление с индекса 0
val = queue.pop(0)  # Dequeue: O(n) - ОЧЕНЬ МЕДЛЕННО
```

---

### Почему `pop(0)` медленный? 🐢

Помните Тему 3 (Массивы)? Массивы хранятся в непрерывной памяти.

Если вы удалите первый элемент (Индекс 0), компьютер создаст "дыру" в начале. Он не может оставить дыру.

* Он должен сдвинуть Индекс 1 на Индекс 0.
* Он должен сдвинуть Индекс 2 на Индекс 1.
* Он должен сдвинуть Индекс 3 на Индекс 2...
* Если в вашей очереди 10,000 элементов, компьютеру придется выполнить 9,999 операций сдвига только для того, чтобы удалить одного человека!

---

### Решение: `collections.deque` 🚀

Python предоставляет специализированный инструмент под названием `deque` (читается как «дек»), что означает **Double-Ended Queue** (Двусторонняя очередь).

Она реализована как Двусвязный Список небольших блоков памяти. Удаление с начала — это просто «удаление головного узла», что происходит мгновенно (O(1)).

```python
from collections import deque

# Инициализация
queue = deque()

# 1. Enqueue (Добавить справа)
queue.append(10)
queue.append(20)
queue.append(30)

# Очередь: [10, 20, 30]

# 2. Dequeue (Удалить слева)
# Обратите внимание на имя функции: popleft
front = queue.popleft()
# Возвращает 10. Очередь: [20, 30]. Стоимость: O(1)

# 3. Peek
print(queue[0])  # Возвращает 20
```

### Когда использовать Очередь?

Если вы слышите:

* «Кратчайший путь»
* «Порядок прибытия»
* «Уровень за уровнем»

Используйте Очередь.

#### 1. Поиск в ширину (BFS) 🌊

Это самый важный алгоритм в теории графов. Представьте, что вы бросили камень в пруд. Рябь расходится слоем за слоем.

* Слой 0: Камень (Старт).
* Слой 1: Вся вода в 1 дюйме от камня.
* Слой 2: Вся вода в 2 дюймах от камня.

Чтобы запрограммировать это исследование "Слой за слоем", вы должны использовать Очередь. Вы обрабатываете всех соседей (Слой 1), потом всех соседей соседей (Слой 2)...

#### 2. Очередь печати 🖨️

Если 5 человек отправляют документы на офисный принтер:

* Принтер помещает их в Очередь.
* Он печатает первый полученный файл первым.
* Неважно, отправил ли CEO файл последним; если стажер отправил файл на 1 минуту раньше, стажер ждет меньше.

## Тема 4: Строки (Strings) 🧵

### Что такое Строка?

Технически, Строка — это просто **Массив символов**.

`"Hello"` — это на самом деле `['H', 'e', 'l', 'l', 'o']`.

Поскольку под капотом это массивы, они обладают той же суперсилой: **Мгновенный доступ (O(1))**. Вы можете получить 3-й символ мгновенно, обратившись к `s[2]`.

---

### Главный подвох: Неизменяемость (Immutability) 🔒

Во многих популярных языках (Python, Java, JavaScript, C#) строки являются **Неизменяемыми (Immutable)**. Это означает, что их нельзя изменить после создания.

Представьте строку как **Каменную Скрижаль**. Вы не можете стереть букву и написать новую. Вы должны высечь совершенно новую плиту.

Если вы попытаетесь изменить символ, компьютер не меняет исходную строку. Он создает совершенно новую копию строки в памяти с учетом изменений.

```python
s = "Hello"

# s[0] = "J"  <-- ❌ ОШИБКА! Камень нельзя изменить.

# Вместо этого вы должны создать НОВУЮ строку:
# Это копирует 'ello', добавляет 'J' и сохраняет в новой
s = "J" + s[1:]
```

---

### Ловушка Конкатенации ⚠️

При `s += " World!"` Python не дописывает буквы в конец старой строки. Он создает в другом месте памяти новую строку «Hello World!» и перенаправляет на нее переменную `s`.

Из-за неизменяемости сложение строк внутри цикла опасно для производительности.

**Плохой код (O(n²)):**

Каждый раз, когда вы делаете `+=`, компьютер создает новую копию.

* "Н"
* "Не" (Копирует "Н", добавляет "е")
* "Hel" (Копирует "Не", добавляет "l")
* ...и так далее.

```python
result = ""

for char in large_list:
    # ❌ НЕВЕРНО! Создает новую копию для КАЖДОГО символа
    result += char
```

**Хороший код (O(n)):**

Используйте **Список (List)**. Списки изменяемы (как маркерная доска, а не камень). Вы собираете все символы и объединяете их один раз в конце.

```python
chars = []

for char in large_list:
    chars.append(char)  # БЫСТРО: Добавление в список

# БЫСТРО: Создает строку только один раз
result = "".join(chars)
```

* **Сложность:** O(k), где k — длина среза.
* **Почему:** Компьютер должен скопировать эти конкретные символы в новое место в памяти. Это не мгновенно.

Проверка, есть ли "cat" внутри "vacation".

* **Сложность:** O(n × m) (Наивный) или O(n) (Оптимизированный).
* **Почему:** В худшем случае нужно проверить каждую начальную позицию в тексте.

## Тема 9: Два Указатели (Two Pointers) 🖐🖐🏼

### Концепция

Паттерн **Two Pointers** — это стратегия оптимизации скорости кода. Она превращает медленное решение «в лоб» (O(n²)) в быстрое линейное решение (O(n)).

---

### Проблема решения «В лоб» (Brute Force)

Представьте, что вы ищете пару носков в куче.

* **В лоб:** Вы берете один носок, затем проверяете каждый другой носок в куче, чтобы узнать, подходит ли он. Затем берете второй носок и снова проверяете все остальные.
* **Сложность:** Вложенные циклы (O(n²)). Если у вас 1000 носков, это 1 000 000 проверок!

Вместо того чтобы сравнивать каждого с каждым, мы используем два индекса (указателя), которые перемещаются по массиву по умному алгоритму.

*(Изображение: Слева енот запутался в нитях — подписан BRUTE FORCE, Справа енот со световым мечом, указывающий на два узла графа — подписан TWO POINTERS)*

---

### Стратегия 1: Сходимость (Встреча посередине) ↔️

Это самая распространенная вариация. Вы ставите один указатель в **Начало** (`left`), а другой в **Конец** (`right`). Они движутся навстречу друг другу, пока не встретятся.

**Требование:** Массив обычно должен быть отсортирован.

#### Пример из жизни: Two Sum II (Sorted)

**Задача:** Найти два числа в отсортированном массиве `[2, 7, 11, 15]`, сумма которых равна 9.

**Логика:** Поскольку массив отсортирован, мы можем принимать решения:

* Сумма слишком маленькая? Нам нужно число побольше. Двигаем `left` вправо (увеличиваем).
* Сумма слишком большая? Нам нужно число поменьше. Двигаем `right` влево (уменьшаем).

```python
def two_sum_sorted(nums, target):
    left = 0
    right = len(nums) - 1

    while left < right:
        current_sum = nums[left] + nums[right]

        if current_sum == target:
            return [left, right]  # Нашли!

        elif current_sum < target:
            # Сумма 9, нам нужно 18.
            # Нужно число побольше. Двигаем левый вперед.
            left += 1

        else:
            # Сумма 20, нам нужно 18.
            # Нужно число поменьше. Двигаем правый назад.
            right -= 1

    return []
```

#### Пример 2: Валидный Палиндром 🪞

Является ли "racecar" палиндромом?

1. Проверяем `left` ('r') и `right` ('r'). Совпадают. Двигаем оба внутрь.
2. Проверяем `left` ('a') и `right` ('a'). Совпадают. Двигаем оба внутрь.
3. В конце концов, они встретятся посередине.

---

### Стратегия 2: Параллельные (Гонка) 🏎️

Здесь оба указателя начинают в начале (`index 0`), но они отслеживают разные вещи.

**Лучше всего для:** Слияния массивов или поиска пересечений.

#### Пример: Слияние двух отсортированных списков

**Дано** `A = [1, 4, 7]` и `B = [2, 3, 6]`. Мы хотим один отсортированный список.

**Логика:**

* Указатель `i` смотрит на A.
* Указатель `j` смотрит на B.
* Сравниваем `A[i]` и `B[j]`.
* Кто меньше? 1 или 2? 1 меньше. Добавляем 1 в наш результат и двигаем ТОЛЬКО указатель `i`.

---

### Стратегия 3: Чтение/Запись (In-Place Modification) ✍️

Это хитрый трюк. Мы используем два указателя на одном массиве.

* **Читатель (j):** Сканирует вперед, чтобы найти новые/валидные элементы.
* **Писатель (i):** Остается позади и перезаписывает массив валидными элементами.

#### Пример: Удаление дубликатов (Sorted)

**Вход:** `[1, 1, 2]` → **Выход:** `[1, 2]`

**Логика:**

1. Читатель видит 1. Это то же самое, что и предыдущее. Игнорируем.
2. Читатель видит 2. Это новое!
3. Писатель пишет 2 рядом с первой 1.

```python
def remove_duplicates(nums):
    # Writer (индекс вставки)
    i = 1

    # Reader (j) сканирует каждое число
    for j in range(1, len(nums)):
        # Если нашли НОВОЕ число (отличное от предыдущего)
        if nums[j] != nums[j - 1]:
            nums[i] = nums[j]  # Копируем его на позицию писателя
            i += 1             # Двигаем писателя вперед

    return i  # Новая длина
```

### Совет для LeetCode 💡

Техника двух указателей обычно требует, чтобы входной массив был отсортирован.

* Если задача просит пару (`a + b = target`), а массив не отсортирован, у вас есть два пути:
  * Использовать **Hash Map** (O(n) время, O(n) память).
  * **Отсортировать массив** (O(n log n) время), затем использовать **Two Pointers** (O(1) память).

## Тема 10: Скользящее Окно (Sliding Window) 🪟

### Концепция

Один из самых элегантных паттернов в разработке алгоритмов. Берет задачу, которая кажется требующей вложенных циклов O(n²), и решает её за один линейный проход O(n).

---

### Аналогия: Картонная Рамка 🏠📜

Представьте длинную бумажную ленту с 1000 чисел. Вы вырезаете прямоугольное отверстие («Окно») в картоне шириной ровно на 3 числа.

1. Вы накладываете рамку на начало: видите `[10, 20, 30]`.
2. Сдвигаете рамку на один шаг вправо.

---

### В чем магия? 🧙‍♂️

Вам не нужно пересчитывать всю сумму заново.

* Число 10 покинуло поле зрения.
* Число 40 вошло в него.
* **Новая Сумма = Старая Сумма – 10 + 40.**

**Цель:** Переиспользовать предыдущую работу вместо того, чтобы начинать с нуля каждый раз. Это превращает медленные вложенные циклы в быстрые линейные сканирования.

---

### Сравнение подходов

| Подход            | Описание                                                                                           | Сложность                                              |
|-------------------|----------------------------------------------------------------------------------------------------|--------------------------------------------------------|
| **«В лоб»**       | Посчитай сумму индексов 0–2. Затем сумму индексов 1–3. Затем 2–4... Вы складываете одни и те же средние числа снова и снова. Расточительство! | O(n²)                                                  |
| **Скользящее Окно** | Возьми предыдущую сумму. Вычти того, кто выпал. Прибавь того, кто вошел.                            | O(n) — Оптимизированный линейный проход                |

---

### Вариация 1: Окно Фиксированного Размера 📏

Используется, когда длина окна задана явно (например, задача *«Найти максимальную сумму любого подмассива размером k»* или «максимум суммы соседних элементов»).

**Пример:** Найти максимальную сумму 3 подряд идущих чисел в массиве `[2, 1, 5, 1, 3, 2]`, k=3.

**Шаги:**

1. **Инициализация:** Считаем сумму первых 3 чисел `[2, 1, 5]`. Сумма = 8.
2. **Сдвиг 1:** Убрать 2, Добавить 1. Окно `[1, 5, 1]`. Новая сумма = 8 - 2 + 1 = 7.
3. **Сдвиг 2:** Убрать 1, Добавить 3. Окно `[5, 1, 3]`. Новая сумма = 7 - 1 + 3 = 9. (**Новый Максимум!**)
4. **Сдвиг 3:** Убрать 5, Добавить 2. Окно `[1, 3, 2]`. Новая сумма = 9 - 5 + 2 = 6.

**Результат: 9.**

```python
def max_sum_fixed(nums, k):
    # 1. Считаем сумму первого окна
    current_sum = sum(nums[:k])
    max_sum = current_sum

    # 2. Сдвигаем окно с индекса k до конца
    for i in range(k, len(nums)):
        # i - это новый элемент, который входит
        # i - k - это старый элемент, который уходит
        current_sum = current_sum - nums[i - k] + nums[i]

        # 3. Обновляем глобальный максимум
        max_sum = max(max_sum, current_sum)

    return max_sum
```

### Вариация 2: Окно Динамического Размера (Акордеон) 🎹

Используется, когда задача просит найти «Самый длинный» или «Самый короткий» подмассив, удовлетворяющий условию. Окно растет и сжимается, как аккордеон или гусеница.

**Стратегия:**

1. **Расширение (Expand):** Двигайте указатель `right`, чтобы включить больше элементов (растяните аккордеон), пока условие не нарушится.
2. **Сжатие (Shrink):** Двигайте указатель `left`, чтобы убрать элементы (сжать аккордеон), пока условие снова не станет верным.

**Пример:** Найти самую длинную подстроку без повторяющихся символов (`"abcabcbb"`).

1. Окно `[a, b, c]`. Валидно. Расширяем (добавляем следующий символ).
2. Окно `[a, b, c, a]`. Невалидно! Дубликат 'a'.
3. Сжимаем: Удаляем слева, пока 'a' не уйдет. Удаляем первый 'a'.
4. Окно `[b, c, a]`. Валидно снова. Расширяем дальше.

Этот процесс похож на игру с аккордеоном: вы растягиваете его (expand), пока всё хорошо, и сжимаете (shrink), пока условие не нарушится.

**Код реализации:**

```python
def length_of_longest_substring(s):
    seen = set()       # Множество для отслеживания символов в окне
    left = 0           # Левый указатель (начало окна)
    max_len = 0        # Максимальная длина найденной подстроки

    # Правый указатель "расширяет" окно
    for right in range(len(s)):

        # Пока наше окно невалидно (есть дубликат), сжимаем слева
        while s[right] in seen:
            seen.remove(s[left])  # Выгоняем символ слева
            left += 1             # Сдвигаем левую границу

        # Добавляем новый символ и обновляем макс. длину
        seen.add(s[right])
        # Размер окна = правый индекс - левый индекс + 1
        max_len = max(max_len, right - left + 1)

    return max_len
```

## Продвинутый Паттерн 2: Поиск по «Пространству Ответов» 🧠

Иногда массив не дается. Вы ищете число, удовлетворяющее условию, а не элемент в списке. Вы запускаете бинарный поиск по диапазону возможных значений ответа.

**Пример:** «Коко ест бананы» (Koko Eating Bananas). Найти минимальную скорость поедания k, чтобы успеть за H часов.

**Логика:**

* Если скорость 1 — слишком медленно.
* Если скорость 1,000,000 — достаточно быстро.
* Ответ находится где-то между 1 и 1,000,000. Запустите Бинарный Поиск по этому диапазону!

```python
def solve_answer_space(low, high):
    ans = high
    while low <= high:
        mid = (low + high) // 2

        if is_possible(mid):  # Вспомогательная функция проверки
            ans = mid         # Эта скорость подходит, попробуем медленнее?
            high = mid - 1
        else:
            low = mid + 1     # Слишком медленно, нужно быстрее

    return ans
```

### Частый Баг: Переполнение (Overflow) 🪱

В таких языках, как C++ или Java, вычисление `mid = (left + right) / 2` может вызвать переполнение целого числа (Integer Overflow), если `left + right` больше максимального размера int (2³¹).

* **Безопасный способ:** `mid = left + (right - left) // 2`
* **Примечание:** Python обрабатывает большие числа автоматически, так что это не обязательно, но интервьюеры это любят.

## Тема 11: Префиксные Суммы (Prefix Sums) +

### Концепция

**Префиксная Сумма** — это техника, работающая как «Машина времени» для массивов. Она позволяет мгновенно отвечать на вопросы об истории данных.

Массив Prefix Sum — это список, где каждый элемент является текущей суммой (накопительным итогом) всех чисел до этой точки.

### Аналогия: Одометр Автомобиля 🚗

Представьте, что вы едете в путешествие. Вы не сбрасываете счетчик пробега каждый день. Вы просто позволяете ему крутиться.

* День 1: Вы проехали 100 км. Одометр: 100.
* День 2: Вы проехали 50 км. Одометр: 150.
* День 3: Вы проехали 200 км. Одометр: 350.

**Вопрос:** Сколько вы проехали в сумме за День 2 и День 3?

* **Медленный способ:** Сложить 50 + 200.
* **Способ Префиксной Суммы:** Взять итог на День 3 (350) и вычесть итог до начала Дня 2 (100).
* **350 - 100 = 250.**

---

### Зачем это использовать?

Представьте, что у вас есть массив из 10 000 чисел (например, ежедневные продажи за 30 лет).
Ваш босс задает 1000 разных вопросов, типа: «Каковы были общие продажи с 500-го по 800-й день?»

#### Способ «В лоб» O(n × m):

Вы пишете цикл от 500 до 800. Затем цикл от 20 до 100...

* **Результат:** Ваш сервер зависает. Выполнять цикл 1000 раз на огромном массиве — слишком медленно.

#### Способ Префиксной Суммы O(1):

Вы вычисляете бегущие суммы один раз. Теперь, чтобы найти сумму между любыми двумя точками, вы делаете всего одно вычитание.

* **Формула:** `Sum(i, j) = Prefix[j] - Prefix[i-1]`
* **Стоимость:** O(1) (Мгновенно).

*(Изображение: Енот в машине: «I don't need to measure the road again. I just check the difference on the meter.»)*

---

### Как это работает (Пошагово)

Найдем сумму подмассива `[2, 5]` (индексы 2 и 3) из массива `[3, 1, 2, 5, 4]`.

#### Шаг 1: Построение Префиксного Массива

Мы проходим по массиву один раз, добавляя текущее число к предыдущей сумме.

* Индекс 0: 3
* Индекс 1: 3 + 1 = 4
* Индекс 2: 4 + 2 = 6
* Индекс 3: 6 + 5 = 11
* Индекс 4: 11 + 4 = 15
* **Prefix = [3, 4, 6, 11, 15]**

#### Шаг 2: Вычисление «Среза»

Нам нужна сумма индексов с 2 по 3 (значения 2, 5).

* **Общая Сумма** (до индекса 3): 11 (Включает 3, 1, 2, 5).
* **Нежелательная История** (до индекса 2): 4 (Это 3, 1).
* **Операция:** 11 - 4 = 7.
* **Проверка:** 2 + 5 = 7. Работает!

*(Изображение слева: Енот бежит «Brute Force Loop». Справа: Енот сидит в кресле «ANSWER: INSTANT! O(1) Math»)*

---

### Трюк с «Фиктивным Нулем» (Dummy Zero) 🛠️

Если вы попытаетесь вычислить `Sum(0, 3)`, вам нужно будет вычесть `Prefix[-1]`. Но `Prefix[-1]` означает конец списка в Python или ошибку в других языках!

Чтобы не проверять «Является ли это первым элементом?», мы добавляем **0** в начало нашего Префиксного Массива. Это действует как «бампер безопасности».

```python
def range_sum_query(nums, queries):
    # 1. Строим Prefix Sum (Размер N + 1)
    # Оригинал: [3, 1, 2]
    # Prefix:   [0, 3, 4, 6]
    prefix = [0] * (len(nums) + 1)

    for i in range(len(nums)):
        prefix[i + 1] = prefix[i] + nums[i]

    # 2. Отвечаем на запросы мгновенно
    results = []
    for start, end in queries:
        # Чтобы получить sum(start, end), берем P[end+1] - P[start]
        # +1 появляется из-за сдвига с фиктивным нулем.
        current_sum = prefix[end + 1] - prefix[start]
        results.append(current_sum)

    return results
```

### Продвинутый Паттерн: Сумма подмассива равна K 🧠

Это классическая задача LeetCode Medium.

**«Найти общее количество подмассивов, сумма которых равна ровно K.»**

Мы не можем использовать технику Sliding Window, потому что массив может содержать отрицательные числа (поэтому сумма не всегда растет). Мы должны использовать **Prefix Sum + Hash Map**.

**Логика:**

Если `Prefix[j] - Prefix[i] = K`, то `Prefix[j] - K = Prefix[i]`.

Мы сохраняем каждую префиксную сумму, которую видим, в Хеш-карту. По мере итерации мы проверяем: «Видел ли я раньше префиксную сумму, равную `Current_Sum - K`?»

```python
def subarray_sum(nums, k):
    count = 0
    curr_sum = 0
    # Map: {prefix_sum : сколько_раз_видели}
    # Инициализируем {0:1} для обработки случаев, когда подмассив начинается с индекса 0
    prefix_map = {0: 1}

    for num in nums:
        curr_sum += num

        # Магическая проверка формулы
        if (curr_sum - k) in prefix_map:
            count += prefix_map[curr_sum - k]

        # Добавляем текущую сумму в карту
        prefix_map[curr_sum] = prefix_map.get(curr_sum, 0) + 1

    return count
```

## Тема 12: Рекурсия (Recursion) 🔄

Рекурсия — это метод, при котором функция вызывает саму себя, чтобы решить меньшую версию задачи.

Думайте об этом как о наборе **Русских Матрёшек**.

1. Вы открываете большую куклу. Внутри кукла поменьше.
2. Вы открываете меньшую куклу. Внутри еще меньше.
3. Вы продолжаете это делать, пока не дойдете до самой крошечной, цельной куклы, которую нельзя открыть.

**Концепция "Начала" (Inception):**

Это решение большой проблемы путем разбиения ее на более мелкие экземпляры той же самой проблемы.

---

### 2 Железных Правила Рекурсии

#### 1. Базовый Случай (Якорь) 🛑

Это "самая крошечная кукла". Это условие, которое останавливает рекурсию.

* **Без этого:** Функция вызывает саму себя вечно (Бесконечный цикл), пока у компьютера не закончится память. Это знаменитая ошибка **Stack Overflow** (Переполнение стека).

#### 2. Рекурсивный Шаг (Действие) 📈

Здесь функция вызывает саму себя с измененными входными данными.

* **Важно:** Входные данные должны приближаться к Базовому Случаю. Если вы вызовете `factorial(n)` внутри `factorial(n)`, вы никогда не сдвинетесь с места. Вы должны вызвать `factorial(n-1)`.

---

### Пример 1: Факториал (!)

**Математика:** 5! = 5 × 4 × 3 × 2 × 1.

**Рекурсивная логика:** 5! = 5 × 4!

```python
def factorial(n):
    # 1. Базовый случай: Знак "Стоп"
    if n <= 1:
        return 1

    # 2. Рекурсивный шаг: Уменьшение задачи
    # "Я пока не знаю ответа. Мне нужно спросить у n-1."
    return n * factorial(n - 1)
```

---

### Как это видит компьютер (Call Stack) 🥞

Помните Тему 7 (Стеки)? Вот где это важно. Когда вы вызываете `factorial(3)`, компьютер не просто знает ответ. Он ставит на паузу текущую функцию и добавляет новый "Список дел" в стек.

#### Фаза 1: Накручивание (Pushing)

1. `factorial(3)`: На паузе. Ждет `factorial(2)`.
2. `factorial(2)`: На паузе. Ждет `factorial(1)`.
3. `factorial(1)`: Базовый случай сработал! Возвращает 1.

#### Фаза 2: Раскручивание (Popping)

4. `factorial(2)` возобновляется: Возвращает 2 × 1 = 2.
5. `factorial(3)` возобновляется: Возвращает 3 × 2 = 6.
6. Готово.

---

### "Прыжок Веры" (Ментальный Хак) 🧠

Не визуализируйте весь стек в голове, используйте **Прыжок Веры**:

1. Предположите, что рекурсивный вызов `factorial(n-1)` магическим образом работает и дает правильный ответ.
2. Сосредоточьтесь только на том, как соединить этот ответ с вашим текущим числом n.

* **Не думайте:** *«Она идет вниз до 1, потом возвращает 1, потом возвращает 2...»*
* **Думайте:** *«Если у меня есть факториал 4, я просто умножаю его на 5.»*

---

### Скрытая Опасность: Фибоначчи и Сложность ⚠️

Посмотрим на последовательность Фибоначчи: 0, 1, 1, 2, 3, 5, 8...

**Формула:** `fib(n) = fib(n-1) + fib(n-2)`

```python
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)
```

**Проблема:** Это чрезвычайно медленно O(2ⁿ).

Чтобы вычислить `fib(5)`, он вычисляет `fib(3)` дважды. Чтобы вычислить `fib(100)`, потребуются триллионы лет.

---

### Решение: Мемоизация (Записная книжка) 📒

Не пересчитывайте. Запоминайте.

Сохраните результат `fib(3)` в Хеш-карте (Словаре). Перед вычислением проверьте карту. Это превращает O(2ⁿ) в O(n).

### Рекурсия vs. Итерация (Циклы)

| Характеристика       | Рекурсия                              | Итерация (Циклы)                 |
|----------------------|---------------------------------------|----------------------------------|
| **Стиль кода**       | Чистый, элегантный, декларативный     | Детальные пошаговые инструкции   |
| **Память**           | Высокая O(N). Использует Стек вызовов | Низкая O(1). Константная память  |
| **Лучше для**        | Деревьев, Графов, Сортировки слиянием  | Простых списков, Счетчиков       |
| **Риск краша**       | Stack Overflow (Переполнение)         | Infinite Loop (Зависание)        |

## Тема 13: Хеш-таблицы (Словари/Dictionaries) 🗝️

Хеш-таблица (или **Hash Map**) — пожалуй, самая важная структура данных для собеседований. Если Массивы — это «Фундамент», то Хеш-таблицы — это «Суперсила».

Это структура данных, которая хранит данные в парах **Ключ-Значение** и обеспечивает **Мгновенный доступ (O(1))**.

---

### Аналогия: Волшебная Библиотека (без порядка) 📚

* **Подход Массива:** Чтобы найти «Гарри Поттера», вам придется пройти вдоль каждой полки, просматривая каждый корешок, пока не найдете книгу. Это займет вечность (O(n)).

* **Подход Хеш-таблицы:** Вы подходите к Волшебному Библиотекарю. Вы говорите «Гарри Поттер». Он щелкает пальцами и телепортирует вас прямо к Ряду 4, Полке 9, Слоту 2. Вы не ищете; вы просто прибываете на место.

---

### «Магия»: Хеш-функция ✨

Как компьютер узнает, куда положить данные, не ища свободное место? Он использует **Хеш-функцию**.

Это математическая формула, которая берет **Вход (Ключ)** и превращает его в **Число (Индекс)**.

* Вход: «Apple» → Формула → Выход: Index 3
* Вход: «Banana» → Формула → Выход: Index 5

Когда вы захотите найти «Apple» позже, компьютер снова запустит формулу: «Apple» → 3. Он идет прямо в слот №3. Он никогда не смотрит на слоты 0, 1 или 2.

---

### Проблема Коллизий 💥

Иногда формула несовершена. Она может случайно выдать один и тот же индекс для двух разных ключей.

* «Apple» → Index 3
* «Grapes» → Index 3

Это называется **Коллизией**.

Если двум людям назначен один и тот же номер в отеле, что происходит?

**Решение (Chaining/Цепочки):** Мы помещаем Связный Список (или «ведро») внутри этого слота.

* Слот 3 содержит: `[Apple]` → `[Grapes]`
* Когда вы ищете «Grapes», компьютер идет в Слот 3, а затем быстро пробегает по короткому списку, чтобы найти нужное.

---

### Два Типа: Map vs. Set

#### 1. Hash Map (Словарь/Dictionary) 📖

Хранит **Ключ → Значение**.

* **Сценарий:** Хранение связей.
* **Пример:** «Мне нужно найти возраст пользователя по его имени.»
* **Код:** `phone_book = {"Alice": 5551234, "Bob": 5559876}`

#### 2. Hash Set (Множество) 🎒

Хранит **Только Ключи**. Без значений.

* **Сценарий:** Проверка существования (Видел ли я это раньше?).
* **Пример:** «Находится ли этот IP-адрес в черном списке?»
* **Код:** `visited_urls = {"google.com", "facebook.com"}`

```python
# Операции с Set в Python
my_set = set()
my_set.add(10)    # O(1)

if 10 in my_set:  # O(1) - Мгновенно!
    print("Нашел")
```

---

### Анализ Сложности

| Операция    | Массив (Неотсорт.) | Хеш-таблица |
|-------------|---------------------|-------------|
| **Вставка** | O(1) (Append)       | O(1)        |
| **Поиск**   | O(n) (Медленно)     | O(1) (Мгновенно) |
| **Удаление**| O(n) (Медленно)     | O(1) (Мгновенно) |

> **Примечание:** В худшем случае (если ВСЕ ключи попадут в коллизию), сложность упадет до O(n), но на интервью мы считаем ее O(1).

---

### Разбор задачи LeetCode: Two Sum 💡

**Задача:** Найти два числа в массиве `[2, 7, 11, 15]`, которые в сумме дают 9.

#### Стратегия «Hash Map» (O(n)):

Вместо двух циклов (O(n²)), мы проходим по списку один раз. Мы используем Map, чтобы «запомнить», что мы видели.

**Трассировка:**

1. **Index 0 (Val 2):** Нам нужно 9 - 2 = 7. Есть ли 7 в карте? Нет. Сохраняем `{2: Index 0}`.
2. **Index 1 (Val 7):** Нам нужно 9 - 7 = 2. Есть ли 2 в карте? **ДА!**
3. **Результат:** Мы нашли пару (2, 7). Возвращаем индексы `[0, 1]`.

```python
def two_sum(nums, target):
    # Map хранит: {Число : Индекс}
    prev_map = {}

    for i, n in enumerate(nums):
        # Видели ли мы эту разницу раньше?
        diff = target - n

        if diff in prev_map:
            return [prev_map[diff], i]

        # Если нет, запоминаем текущее число на будущее
        prev_map[n] = i
```

### Совет для LeetCode 🧠

Если задача спрашивает о:

1. **"Частоте"** (Подсчитать, сколько раз встречается x) ⇒ Используйте Hash Map `{Item: Count}`.
2. **"Дубликатах"** (Проверить, существует ли x) ⇒ Используйте Hash Set.
3. **"Парах"** (Найти x и y) ⇒ Используйте Hash Map для хранения «дополнения» (Target - x).

> **Факты о хеш-таблицах:**
> 1. Длина всех хэш-ключей одинакова.
> 2. Индекс — это остаток от ключа.
> 3. При коллизии создаётся бакет для хранения совпадений по хэшу (связный список).

## Тема 14: Бинарный Поиск (Binary Search / Divide & Conquer) ✂️

### Концепция

Бинарный Поиск — это самый эффективный способ найти элемент в Отсортированном списке. Он работает путем многократного деления области поиска пополам.

---

### Основной Алгоритм (Итеративный)

Мы используем три указателя:

1. **Left (l):** Начало текущего диапазона.
2. **Right (r):** Конец текущего диапазона.
3. **Mid (mid):** Середина.

```python
def binary_search(nums, target):
    left, right = 0, len(nums) - 1

    while left <= right:
        # Находим середину
        mid = (left + right) // 2

        # Случай 1: Нашли!
        if nums[mid] == target:
            return mid

        # Случай 2: Цель в правой половине
        elif nums[mid] < target:
            left = mid + 1  # Сдвигаем начало вперед

        # Случай 3: Цель в левой половине
        else:
            right = mid - 1  # Сдвигаем конец назад

    return -1  # Не найдено
```

---

### Главное Требование ⚠️

Бинарный Поиск работает **ТОЛЬКО** если массив **Отсортирован**.

* `[1, 3, 5, 9, 12]` ✅ Работает.
* `[5, 1, 9, 3, 12]` ❌ Не сработает. Сначала нужно отсортировать O(n log n) или использовать Линейный Поиск O(n).

---

### Сложность: Сила Логарифмов 📉

Мы называем это **Логарифмическим Временем O(log n)**.

Это невероятно быстро, потому что количество шагов растет очень медленно по сравнению с данными.

---

### Продвинутый Паттерн 1: Поиск Границ (Первое/Последнее вхождение)

Часто LeetCode не спрашивает «Есть ли здесь 5?». Он спрашивает **«Где первая 5?»** или **«Где последняя 5?»**.

* **Вход:** `[1, 2, 5, 5, 5, 9]`
* **Стандартный бинарный поиск** может вернуть индекс 3 (средняя 5). Нам нужен индекс 2 (первая 5).

**Техника:** Когда вы находите цель, не останавливайтесь. Сохраните индекс и продолжайте поиск в нужной половине (влево для первой, вправо для последней).

```python
def find_first_occurrence(nums, target):
    left, right = 0, len(nums) - 1
    result = -1

    while left <= right:
        mid = (left + right) // 2

        if nums[mid] == target:
            result = mid       # Записываем потенциальный ответ
            right = mid - 1    # Продолжаем искать СЛЕВА

        elif nums[mid] < target:
            left = mid + 1
        else:
            right = mid - 1

    return result
```

## Ключевая Терминология (Семейное Древо) 👥

* **Root** (Корень/Предок): Верхний узел.
* **Node** (Узел): Круг, содержащий данные.
* **Child** (Ребенок): Узел, подключенный ниже.
* **Parent** (Родитель): Узел, находящийся выше.
* **Leaf** (Лист): Узел, не имеющий детей (Дно/Конец дерева).
* **Subtree** (Поддерево): Любой узел и все его потомки образуют "маленькое дерево" внутри большого.

## Тема 17: Бинарные Деревья

Бинарное Дерево — это особый тип дерева со строгим правилом:

**У каждого узла может быть НЕ БОЛЕЕ ДВУХ детей:**

1. Левый Ребенок (Left Child)
2. Правый Ребенок (Right Child)

### Структура Кода

Мы определяем узел как в Двусвязном Списке:

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val       # Данные
        self.left = left     # Указатель на левого ребенка
        self.right = right   # Указатель на правого ребенка
```

---

### Как "ходить" по Дереву (Обходы DFS) 🚶‍♂️

Вы не можете использовать простой цикл `for`, потому что дерево ветвится. Мы используем **Рекурсию** для исследования дерева. Существует 3 основных способа посетить каждый узел (Поиск в глубину).

#### 1. Pre-order Traversal (Я, Лево, Право)

**Логика:** "Я обрабатываю себя сначала, затем иду к левому ребенку, затем к правому."

* **Порядок:** Root → Left → Right
* **Сценарий:** Клонирование дерева (нужно создать родителя, прежде чем создавать детей).

```python
def preorder(root):
    if not root:
        return

    print(root.val)       # 1. ОБРАБОТАТЬ СЕБЯ
    preorder(root.left)   # 2. ИДТИ ВЛЕВО
    preorder(root.right)  # 3. ИДТИ ВПРАВО
```

#### 2. In-order Traversal (Лево, Я, Право)

**Логика:** "Я иду до упора влево сначала. Я обрабатываю себя, когда возвращаюсь. Затем иду вправо."

* **Порядок:** Left → Root → Right
* **Сценарий:** Бинарные Деревья Поиска (BST). Этот обход печатает числа в отсортированном порядке!

```python
def inorder(root):
    if not root:
        return

    inorder(root.left)    # 1. ИДТИ ВЛЕВО
    print(root.val)       # 2. ОБРАБОТАТЬ СЕБЯ
    inorder(root.right)   # 3. ИДТИ ВПРАВО
```

#### 3. Post-order Traversal (Лево, Право, Я)

**Логика:** "Я посещаю всех своих детей сначала. Себя я обрабатываю последним."

* **Порядок:** Left → Right → Root
* **Сценарий:** Удаление дерева (вы должны удалить детей, прежде чем сможете удалить родителя) или вычисление размера папки.

```python
def postorder(root):
    if not root:
        return

    postorder(root.left)   # 1. ИДТИ ВЛЕВО
    postorder(root.right)  # 2. ИДТИ ВПРАВО
    print(root.val)        # 3. ОБРАБОТАТЬ СЕБЯ
```

---

### Основные Алгоритмы Деревьев по паттерну: Доверяй Рекурсии

#### 1. Максимальная глубина (Max Depth)

**Задача:** Сколько слоев в глубину у дерева?

* **Базовый случай:** Если узел `None`, глубина 0.
* **Рекурсивный шаг:** Моя глубина = 1 + max(глубина_слева, глубина_справа).

```python
def maxDepth(root):
    if not root:
        return 0

    left_h = maxDepth(root.left)
    right_h = maxDepth(root.right)

    return 1 + max(left_h, right_h)
```

#### 2. Одинаковые Деревья (Same Tree)

**Задача:** Выглядят ли два дерева (`p` и `q`) абсолютно одинаково?

**Логика:**

* Если оба `None` ⇒ `True`.
* Если одно `None` (а другое нет) ⇒ `False`.
* Если значения разные ⇒ `False`.
* Иначе, проверь совпадает ли `left` И совпадает ли `right`.

```python
def isSameTree(p, q):
    if not p and not q:
        return True
    if not p or not q:
        return False
    if p.val != q.val:
        return False

    return isSameTree(p.left, q.left) and isSameTree(p.right, q.right)
```

#### 3. Инвертировать Дерево (Invert Binary Tree) 🤔

**Задача:** Поменяйте местами каждого левого и правого ребенка.

**Логика:** Поменяй моих детей, затем скажи им поменять своих детей.

```python
def invertTree(root):
    if not root:
        return None

    # Swap
    root.left, root.right = root.right, root.left

    # Рекурсия
    invertTree(root.left)
    invertTree(root.right)

    return root
```

#### 4. Сумма Пути (Path Sum)

**Задача:** Есть ли путь от Корня до Листа, который в сумме дает `targetSum`?

**Логика:** Вычти мое значение из цели. Спроси детей: "Можете ли вы собрать новую цель?"

```python
def hasPathSum(root, targetSum):
    if not root:
        return False

    # Проверка: Я Лист И я завершаю сумму
    if not root.left and not root.right:
        return targetSum == root.val

    new_target = targetSum - root.val
    return hasPathSum(root.left, new_target) or hasPathSum(root.right, new_target)
```

---

### Анализ Сложности 📈

* **Время:** O(n). Почти в каждой задаче с деревом вы посещаете каждый узел ровно один раз.
* **Память:** O(h) (Высота дерева).
  * **Лучший случай (Сбалансированное):** Дерево похоже на треугольник. Высота log n. Память O(log n).
  * **Худший случай (Скошенное):** Дерево — прямая линия (Связный Список). Высота n. Память O(n) из-за стека рекурсии.

---

### Обход по Уровням (BFS) 🌊

Иногда нужно идти "Уровень за Уровнем" (Уровень 0, затем Уровень 1, затем Уровень 2).

Рекурсия (DFS) для этого плоха. Используйте **Очередь** (Тема 8)!

---

### Пример реализации кода

```python
from collections import deque

def levelOrder(root):
    if not root:
        return []
    queue = deque([root])
    results = []

    while queue:
        level = []
        for i in range(len(queue)):
            node = queue.popleft()
            level.append(node.val)
            if node.left:
                queue.append(node.left)
            if node.right:
                queue.append(node.right)
        results.append(level)

    return results
```

### Бинарное дерево поиска - визуал

* Количество потомков всегда два.
* Левый потомок всегда меньше родителя.
* Правый потомок всегда больше родителя.

### Бинарное дерево - добавление элемента

1. Сравнение с значением корня.
2. Выбор стороны в ветке.
3. Проверка каждого узла < / >.
4. Пройти всю глубину и разместить.

**Реализация такая же как в связных узлах:**

* Ссылки на узлы вперёд/назад = ссылки на узлы лево/право.

## Тема 18: Бинарные Деревья Поиска (BST)

### Что делает дерево "Поисковым"?

Обычное бинарное дерево — это просто куча данных. Если вы хотите найти число 42, вам придется проверить каждый узел (O(n)), потому что оно может быть где угодно.

**Бинарное Дерево Поиска (BST)** — это высокоорганизованная структура. Оно следует строгому правилу, которое позволяет вам действовать как снайпер: вы можете отсекать половину оставшихся данных на каждом шаге.

### Золотое Правило 📜

**Для каждого узла в дереве:**

1. **Левое поддерево:** ВСЕ значения должны быть **меньше** узла.
2. **Правое поддерево:** ВСЕ значения должны быть **больше** узла.

> **Примечание:** Это должно быть верно для **каждого** узла, а не только для корня!

---

### Поиск в BST 🔍

Благодаря Золотому Правилу нам никогда не нужно «угадывать», куда идти.

**Сценарий:** Найти 12 в дереве с корнем 10.

1. Сравниваем 12 с Корнем (10).
2. 12 > 10? Да.
3. **Действие:** Идем Вправо. **Важно:** Мы мгновенно игнорируем все Левое поддерево. Мы знаем, что 12 там быть не может.
4. Следующий узел 15. Сравниваем 12 с 15.
5. 12 < 15? Да.
6. **Действие:** Идем Влево. Нашли 12!

**Реализация в коде:**

```python
def search_bst(root, val):
    # Базовый случай: Не нашли или Нашли!
    if not root or root.val == val:
        return root

    # Время выбора
    if val < root.val:
        return search_bst(root.left, val)  # Идем Влево
    else:
        return search_bst(root.right, val)  # Идем Вправо
```

---

### Вставка в BST ➕

Вставка — это просто Поиск, который заканчивается созданием нового узла.

**Цель:** Вставить 12 в дерево.

1. Ищите 12.
2. Когда вы попадете в `None` (пустое место), где 12 должно быть, создайте там узел.

**Код вставки:**

```python
def insert_bst(root, val):
    if not root:
        return TreeNode(val)  # Создаем узел здесь

    if val < root.val:
        root.left = insert_bst(root.left, val)
    elif val > root.val:
        root.right = insert_bst(root.right, val)

    return root
```

---

### Ловушка "Валидации" ⚠️

Самая популярная задача на интервью — **"Validate Binary Search Tree"**.

* **Ошибка новичка:** Проверка только непосредственных детей.
  * `if root.left.val < root.val < root.right.val ...`

* **Почему это не работает:** Посмотрите на это невалидное дерево:
  * Корень: 10
  * Правый ребенок: 15 (Верно > 10)
  * Правый-Левый внук: 6
  * **Ошибка:** 6 находится справа от 10, значит, оно должно быть больше, но это не так. Проверка только родителя 15 не поймает это глобальное нарушение.

**Правильное решение (Передача диапазона):**

Каждый узел должен вписываться в определенный диапазон `(min, max)`.

* Корень: `(-inf, +inf)`
* Идем Влево? Максимум становится значением родителя: `(-inf, 10)`
* Идем Вправо? Минимум становится значением родителя: `(10, +inf)`

**Код проверки валидности:**

```python
def isValidBST(root):
    def validate(node, lower_limit, upper_limit):
        if not node:
            return True

        # Проверяем, нарушены ли строгие границы
        if not (lower_limit < node.val < upper_limit):
            return False

        # Влево: ограничиваем верх. Вправо: ограничиваем низ.
        return (validate(node.left, lower_limit, node.val) and
                validate(node.right, node.val, upper_limit))

    return validate(root, float("-inf"), float("inf"))
```

---

### В чем разница BT & BST?

* **BST** — это инструмент для быстрого поиска, вставки и удаления отсортированных данных.
* **Обычное бинарное дерево** — это каркас. На его базе строятся не только BST, но и кучи, деревья отрезков (Segment Trees), синтаксические деревья и многое другое.

---

### Анализ Сложности 📈

| Операция | Сбалансированное BST | Скошенное BST (Линия) |
|----------|----------------------|------------------------|
| **Поиск**  | O(log n)             | O(n)                   |
| **Вставка**| O(log n)             | O(n)                   |
| **Удаление**| O(log n)            | O(n)                   |

---

### Что такое Скошенное Дерево?

Если вы вставляете числа по порядку `1, 2, 3, 4, 5`, каждое число идет вправо. Дерево превращается в **Связный Список**.

**Решение:** Реальные системы используют **Самобалансирующиеся Деревья** (AVL, Red-Black), чтобы автоматически вращаться и сохранять высоту log n. Вам не нужно кодить их на интервью, просто знайте, что они существуют.

---

### Совет для LeetCode 💡

**Магия In-Order Traversal:**

Если вы выполните **In-Order Traversal** (Лево → Корень → Право) на BST, результат будет **Отсортированным Массивом**.

Это самый простой способ решить задачу «Найти K-й наименьший элемент в BST». Просто сделайте обход и остановитесь на К-м числе!

# Тема 19: Кучи (Приоритетные Очереди) ⛰️

## Что такое Куча?

**Куча (Heap)** — это специализированная древовидная структура данных, которая удовлетворяет **Свойству Кучи**.

В отличие от Бинарного Дерева Поиска (которое полностью отсортировано слева направо), Куча отсортирована лишь **частично (сверху вниз)**. Её волнует только одно: **Кто на вершине?**

---

## Два Вкуса 🍦

1.  **Min-Heap:** Самое **маленькое** число всегда в Корне. Каждый родитель меньше своих детей.
2.  **Max-Heap:** Самое **большое** число всегда в Корне. Каждый родитель больше своих детей.

---

## Аналогия: Отделение Неотложной Помощи 🚑

Представьте приемный покой больницы.

*   **Очередь (FIFO):** Первый пришел — первый обслужен. Пациент с порезом пальца, пришедший в 8:00, попадает к врачу раньше пациента с сердечным приступом, пришедшего в 8:05. **Это плохо.**
*   **Приоритетная Очередь (Heap):** Пациентов лечат на основе **тяжести состояния**. Пациент с сердечным приступом мгновенно «всплывает» в начало очереди, независимо от времени прибытия.

---

## Операции и Скорость ⚡️

**Ключевая цель Куч:** Быстрый доступ к экстремальному элементу (Мин или Макс).

*   **Операция: Peek**
    *   Сложность: O(1)
    *   Описание: Мгновенно. «VIP» всегда наверху.

*   **Операция: Push**
    *   Сложность: O(\log n)
    *   Описание: Добавить внизу, затем «всплыть» на нужное место.

*   **Операция: Pop**
    *   Сложность: O(\log n)
    *   Описание: Удалить верх, переместить низ наверх, затем «погрузить».

*   **Операция: Search**
    *   Сложность: O(n)
    *   Описание: Медленно! Кучи НЕ предназначены для поиска произвольных элементов.

---

## Модуль `heapq` в Python 🐍

В Python есть встроенный модуль `heapq`.  
**Важное замечание:** По умолчанию он реализует **Min-Heap**.

```python
import heapq

# 1. Создание кучи

nums = [5, 1, 9, 3]
heapq.heapify(nums)

# nums теперь переорганизован за O(n).

# Самый маленький элемент под индексом 0.
# Структура: [1, 3, 9, 5]

# 2. Получить минимальный (Peek)

print(nums[0]) # Вернет 1. O(1)

# 3. Добавить элемент (Push)

heapq.heappush(nums, 2) # Добавляет 2. O(log n)

# 4. Удалить минимальный (Pop)

smallest = heapq.heappop(nums)

# Вернет 1. Куча перебалансируется. O(log n)
```

---

## Хак: Как получить Max-Heap? 🤔

Поскольку в Python нет класса MaxHeap, мы используем математический трюк: **Умножаем на -1.**

*   Оригинал: `[10, 20, 5]`
*   Храним: `[-10, -20, -5]`
*   Самый маленький — `-20`. Когда вы достаете его (pop), умножьте на -1 снова, чтобы получить 20.

```python
max_heap = []

heapq.heappush(max_heap, -10)
heapq.heappush(max_heap, -20)
heapq.heappush(max_heap, -5)

# Получить самый большой

largest = -1 * heapq.heappop(max_heap) # Вернет 20
```

### Реальные сценарии использования

1. **Планировщики задач:** Процессор выбирает задачу с наивысшим приоритетом для запуска.
2. **Алгоритм Дейкстры:** Поиск кратчайшего пути в графе требует всегда выбирать «ближайший» узел следующим.
3. **Слияние K отсортированных списков:** Объединение нескольких потоков данных (например, логов).

---

### Стратегия LeetCode: Top K Elements 🏆

**Задача:** «Найти K-й самый большой элемент в потоке/массиве.»

#### Подход 1: Сортировка (Наивно)
Отсортировать весь массив.
• Цена: **O(N log N)**. Слишком медленно для больших данных.

#### Подход 2: Min-Heap (Ход Профи)
Поддерживайте Min-Heap размером ровно **K**.

• Итерируясь, добавляйте числа в кучу.
• Если размер кучи стал **> K**, удалите самое маленькое (корень).
• **Логика:** Если вы храните только **K самых больших чисел**, то «самое маленькое» из этих **K** (корень) будет ровно **K-м самым большим** в общем списке.
• Цена: **O(N log K)**. Г much быстрее, когда **K** мало.

```python
def findKthLargest(nums, k):
    heap = []

    for n in nums:
        heapq.heappush(heap, n)

        if len(heap) > k:
            heapq.heappop(heap) # Удаляем самое маленькое из группы

    return heap[0] # Корень — это K-й самый большой
```

---

### 4 Важных Совета 💡

1. **«К-й Самый Маленький?»** Используйте Max-Heap (храните отрицательные). Размер K.
2. **«К-й Самый Большой?»** Используйте Min-Heap. Размер K.
3. **«Медиана потока?»** Используйте ДВЕ кучи! Max-Heap для левой половины (маленькие числа) и Min-Heap для правой (большие).
4. **Не ищите:** Никогда не проверяйте наличие `x` в куче через `if x in heap`. Это **O(N)**. Используйте Set, если нужен поиск.

---

### 3 Кодерских Хака 🛠️

1. **heapify быстр:** `heapq.heapify(list)` работает за **O(N)**, а не **O(N log N)**. Используйте для инициализации.
2. **heappushpop:** Если нужно добавить и сразу удалить, используйте `heapq.heappushpop(h, item)`. Это эффективнее двух вызовов.
3. **Кортежи работают:** Можно хранить кортежи `(приоритет, имя_задачи)`. Python сравнивает первый элемент. Отлично для объектов.

```python
# Сортировка кортежей

tasks = []

heapq.heappush(tasks, (1, "Критический Баг"))
heapq.heappush(tasks, (3, "Писать Доки"))
heapq.heappush(tasks, (2, "Рефакторинг"))

# Удалит (1, "Критический Баг") первым, т.к. 1 < 2 < 3

print(heapq.heappop(tasks))
```
