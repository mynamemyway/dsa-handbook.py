# 📘 SQL: Конспект синтаксиса и логики (Техническая выжимка)

## 🔑 1. Словарь операторов и ключевых слов
- **AS** — Присваивает временный алиас столбцу, таблице или результату выражения.
- **CASE** — Условный оператор. Аналог `if-elif-else`. Позволяет возвращать разные значения в зависимости от условий.
- **WHEN** — Задает условие проверки внутри блока `CASE`.
- **THEN** — Указывает значение, которое вернется, если условие `WHEN` истинно.
- **ELSE** — Значение по умолчанию, если ни одно условие `WHEN` не выполнено.
- **END** — Завершает блок `CASE`.
- **SELECT** — Извлекает данные: выбирает столбцы, вычисляемые выражения или агрегаты.
- **FROM** — Определяет источник данных (таблицу, представление или подзапрос).
- **WHERE** — Фильтрует строки на этапе выборки до выполнения группировки или сортировки.
- **BY** — Ключевое слово, используемое в составных операторах (`GROUP BY`, `ORDER BY`, `PARTITION BY`) для указания критерия обработки.
- **GROUP** — Группирует строки с одинаковыми значениями для применения агрегатных функций (`COUNT`, `SUM`, `AVG` и др.).
- **HAVING** — Фильтрует результаты после группировки. Аналог `WHERE`, но работает с агрегатными функциями.
- **ORDER** — Сортирует итоговый набор данных по одному или нескольким столбцам (`ASC`/`DESC`).
- **PARTITION** — Логически разделяет набор данных на группы для выполнения оконных функций.
- **UNION** — Объединяет результаты двух или более `SELECT`-запросов в один набор. Автоматически удаляет дубликаты.
- **OVER** — Определяет окно для оконных функций. Задает границы (`PARTITION BY`) и порядок обработки (`ORDER BY`) без сворачивания строк.
- **LIKE** — Оператор сравнения строк с использованием шаблонов (`%` — любое количество символов, `_` — один символ).
- **BETWEEN** — Проверяет вхождение значения в указанный диапазон (включительно: `value >= min AND value <= max`).
- **UPDATE** — Изменяет существующие данные в таблице.
- **SET** — Назначает новые значения столбцам в операторе `UPDATE`.
- **INSERT** — Добавляет новые строки в таблицу.
- **DELETE** — Удаляет строки из таблицы по заданному условию.
- **DROP** — Полностью удаляет объект базы данных (таблицу, схему, индексы, БД) вместе со структурой и данными. Действие необратимо.

## 🔗 2. Типы JOIN (Объединение таблиц)
- **JOIN (INNER JOIN)** — Возвращает только те строки, для которых найдено совпадение в обеих таблицах.
- **LEFT JOIN** — Возвращает все строки из левой таблицы и совпадающие строки из правой. При отсутствии совпадений поля правой таблицы заполняются `NULL`.
- **RIGHT JOIN** — Возвращает все строки из правой таблицы и совпадающие строки из левой. При отсутствии совпадений поля левой таблицы заполняются `NULL`.
- **LEFT OUTER JOIN** — Полный синоним `LEFT JOIN`.
- **CROSS JOIN** — Возвращает декартово произведение таблиц: каждая строка левой таблицы комбинируется с каждой строкой правой.

## 🔑 3. Ключи, связи и отношения (Relation)
- **PRIMARY KEY** — Первичный ключ. Уникально идентифицирует каждую строку в таблице. Гарантирует отсутствие `NULL` и дубликатов.
- **FOREIGN KEY** — Внешний ключ. Устанавливает ссылочную связь с `PRIMARY KEY` другой таблицы. Обеспечивает целостность данных.
- **RELATION** — Отношение (связь). Логическое соединение данных между таблицами, реализуемое через пару `PRIMARY KEY` → `FOREIGN KEY`.

## 🗄️ 4. Совместимость СУБД и Синтаксис
> Изначально приучайте себя писать максимально переносимый код, как от одной базы к другой, так и от человека к человеку... В основном Вам придётся сталкиваться с тремя СУБД - MySQL, SQLite(3) и PostgreSQL. Почему я не упомянул про Мелкософтный SQL - он не бесплатный, и чем дальше, тем реже используется. Что касается MySQL и SQLite(3) - то синтаксис очень похож и различия в них минимальны, даже не столько в синтаксисе, сколько в возможностях, а вот PostgreSQL придерживается традицией.

### Примеры создания таблицы и вставки данных
**MySQL**
```sql
CREATE TABLE author (
    `author_id` INT PRIMARY KEY AUTO_INCREMENT,
    `author_name` VARCHAR(30) NOT NULL
);
INSERT INTO author (`author_name`) VALUES ('Достоевский Ф.М.');
```
**SQLite3**
```sql
CREATE TABLE author (
    `author_id` INTEGER PRIMARY KEY AUTOINCREMENT,
    `author_name` VARCHAR(30) NOT NULL
);
INSERT INTO author (`author_name`) VALUES ('Достоевский Ф.М.');
```
**SQLite before v3.0**
```sql
CREATE TABLE author (
    `author_id` INTEGER PRIMARY KEY,
    `author_name` VARCHAR(30) NOT NULL
);
INSERT INTO author (`author_id`, `author_name`) VALUES (NULL, 'Достоевский Ф.М.');
```
**PostgreSQL**
```sql
CREATE TABLE author (
    author_id SERIAL PRIMARY KEY,
    author_name VARCHAR(30) NOT NULL
);
INSERT INTO author (author_name) VALUES ('Достоевский Ф.М.');
```

📦 **Результат выполнения этих запросов каждого к своей базе будет идентичным:**
- Будет создана таблица `author` с двумя полями `author_id` (целочисленный уникальный идентификатор, автоматически увеличивающийся на 1, одновременно являющийся первичным ключом) и `author_name` (строка размером 30 символов, которая не может принимать значение NULL)
- Будет добавлена запись: `| 1 | 'Достоевский Ф.М.' |`

## 📏 5. Правила оформления, кавычки и переносимость
### Обособление имён полей
Касательно названия полей - обязательно обособление полей только если:
1. Начинается с цифры;
2. Присутствуют отличные от букв, цифр, № и нижнего подчёркивания символы (в том числе и пробел)

⚠️ Однако, в MySQL и SQLite такое исполнение считается устаревшим, т.к. на данный момент уже зарезервирован ряд слов в качестве операторов, к примеру: `user`, `name`, `id`... Легче в редакторе кода избавиться от них в случае необходимости элементарной заменой на нулевой символ (`'\0'`, `'\u0000'`, `'\x00'`). 
💡 Что касается строкового содержания, у всех оно заключается в одинарные кавычки!

### Универсальный запрос (работает на MySQL, SQLite(3) и PostgreSQL)
```sql
SELECT
    row_number() over () as №№,
    'Книга года 2020' AS Номинация,
    title AS Название,
    year_publication AS Год_издания
FROM book
WHERE year_publication = 2020
ORDER BY №№, Название
```

### Специальные символы и кавычки в разных СУБД
В случае если требуется наличие каких-то специальных символов, тогда обособление поля - необходимость, причём в MySQL и SQLite(3) можно обособлять название поля апострофом, одинарными, двойными кавычками, а в PostgreSQL только двойными кавычками, при этом, в PostgreSQL эти кавычки будут присутствовать в названии поля:

```text
+----+-----------------+-----------------------+---------------+
| №№ | Номинация       | Название              | "Год издания" |
+----+-----------------+-----------------------+---------------+
```

### Визуальное форматирование
Визуально разделяйте отдельные блоки кода пробелами (табуляцией):
```sql
WITH zerro_amount AS (
        SELECT
            DISTINCT
            `author_id`,
            `author_name`,
            COUNT(`book_id`) OVER w AS `amount`
        FROM author a
            LEFT JOIN book_author USING (`author_id`)
            LEFT JOIN book USING(`book_id`)
        WHERE `available_numbers` IS NULL OR `available_numbers` = 0
        WINDOW w AS (PARTITION BY `author_id` ORDER BY `author_id`)
    )
SELECT
    `author_name` AS `Автор`,
    (CASE amount > 0
        WHEN 1 THEN amount
        ELSE 'нет'
    END) AS `Количество`
FROM zerro_amount za
ORDER BY `author_name`
```

### Комментарии и отладка
Старайтесь максимально описывать свой код в комментариях:

```sql
/* С РЕЗУЛЬТАТОМ 'zerro_amount'  */
WITH zerro_amount AS (
    /* предварительный запрос */
        /* ВЫБРАТЬ */
        SELECT
            DISTINCT /* уникальные строки */
            `author_id`,    /* идентификатор автора */
            `author_name`,  /* имя автора */
            COUNT(`book_id`) OVER w AS `amount` /* всего произведений автора */
        /* ИЗ */
        FROM author a /* author_id, author_name */
            LEFT JOIN book_author USING (`author_id`)   /* book_id */
            LEFT JOIN book USING(`book_id`)             /* available_numbers */
        /* В КОТОРЫХ: все книги на руках, либо произведений автора в библиотеке нет */
        WHERE `available_numbers` IS NULL OR `available_numbers` = 0
        /* ОКНО: по идентификатору автора */
        WINDOW w AS (PARTITION BY `author_id` ORDER BY `author_id`)
    )
/* основной запрос */
/* ВЫБРАТЬ: */
SELECT 
    `author_name` AS `Автор`,
    (CASE amount > 0    /* есть ли книги в библиотеке? */
        WHEN 1 THEN amount  /* 1 - истина, есть */
        ELSE 'нет'          /* 0 - ложь, нет */
    END) AS `Количество`
/* ИЗ: */
FROM zerro_amount za    /* из предварительного запроса */
/* ПОРЯДОК(СОРТИРОВКА) ПО: */
ORDER BY `author_name`
```

### Пробелы в скобках
Вставляйте один пробельный символ - `" "` между открывающей скобкой `"("` и командой или другой такой-же скобкой:
```sql
SELECT
...
    ROUND( DATEDIFF( (CASE ... THEN blabla1 ... ELSE blabla2 END), 2) AS `что-то-там`
    ...
FROM ...
WHERE ...
```

### Проверьте, что последовательность команд указана верно (она отличается от последовательности выполнения команд в запросе):

- **SELECT** 'столбцы или * для выбора всех столбцов; обязательно'
- **FROM** 'таблица; обязательно'
- **WHERE** 'условие/фильтрация, например, city = 'Moscow'; необязательно'
- **GROUP BY** 'столбец, по которому хотим сгруппировать данные; необязательно'
- **HAVING** 'условие/фильтрация на уровне сгруппированных данных; необязательно'
- **ORDER BY** 'столбец, по которому хотим отсортировать вывод; необязательно'
